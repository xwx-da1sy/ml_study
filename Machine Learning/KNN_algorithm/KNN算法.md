## 第一节 KNN算法简介

KNN，全称是 `K-Nearest Neighbors`，中文通常叫 **K 近邻算法**。

它是一种比较直观的机器学习算法：当我们要判断一个新样本属于哪一类时，不是先训练出一条复杂的公式，而是去训练集中找和它“最像”的若干个样本，然后根据这些邻居的结果来做判断。

一句话理解：

```text
看离我最近的 K 个样本大多数属于哪一类，我就把当前样本判断为哪一类。
```

例如判断一部电影是爱情片还是动作片，可以用“打斗镜头次数”和“接吻镜头次数”作为特征。现在来了一部新电影，我们就计算它和已有电影之间的距离，找出距离最近的 K 部电影。如果这 K 部电影里爱情片更多，那么新电影就更可能被判断为爱情片。

### 1. KNN的基本思想

KNN 的核心思想可以拆成三步：

1. 准备已经有标签的数据集。
2. 计算新样本和训练集中每个样本之间的距离。
3. 找出距离最近的 K 个样本，并根据它们的标签进行投票。

如果是分类问题，通常看 K 个邻居中哪个类别最多。

如果是回归问题，也可以取 K 个邻居目标值的平均值，得到一个连续数值预测结果。

### 2. K值的含义

这里的 `K` 表示参与判断的最近邻样本数量。

- `K = 1`：只看离新样本最近的 1 个样本，结果容易受噪声影响。
- `K` 较大：判断会更稳定，但也可能把距离较远、关系没那么强的样本算进来。

所以 KNN 中的 `K` 不是模型自动学出来的参数，而是我们需要提前设置的超参数。实际使用时，通常会通过测试集或交叉验证来选择一个表现较好的 K 值。

### 3. K的选择与模型复杂度

K 的大小会直接影响 KNN 模型的复杂度。

这里说的“模型复杂度”，不是指 KNN 有很多参数或者公式很复杂。KNN 本身几乎没有训练过程，它的复杂度主要体现在：模型对训练数据局部细节的敏感程度。

当 `K` 过小时，例如 `K = 1`，模型只参考最近的一个样本。这样做会非常依赖训练集中的局部点。

如果训练集中刚好有噪声点、异常点，或者某个样本的位置比较特殊，模型就可能被这个单独的样本影响，做出不稳定的判断。

这时模型会把训练数据中的细节甚至噪声都记下来，决策边界往往会变得非常曲折。它在训练集上可能表现很好，但遇到新的测试样本时泛化能力差，这就是 **过拟合**。

可以理解为：

```text
K 过小 -> 太相信局部邻居 -> 对噪声敏感 -> 模型复杂 -> 容易过拟合
```

当 `K` 过大时，模型会参考大量邻居，甚至把离当前样本较远、关系不大的样本也算进来。

这样模型的判断会变得过于平滑，局部差异被忽略。它不太容易被个别噪声影响，但也可能学不到数据中真实存在的细节规律。

这时模型在训练集和测试集上都可能表现一般，属于 **欠拟合**。

可以理解为：

```text
K 过大 -> 参考范围太广 -> 忽略局部规律 -> 模型简单 -> 容易欠拟合
```

所以选择 K 值时，本质上是在做一个平衡：

- K 小：模型更灵活，方差较大，容易过拟合。
- K 大：模型更平滑，偏差较大，容易欠拟合。

实际使用时，通常不会只凭感觉选 K，而是尝试多个 K 值，然后通过验证集、测试集或交叉验证来比较模型效果。

### 4. 距离度量

KNN 之所以能判断“谁和谁更像”，关键就在于距离计算。

常见距离有：

- 欧氏距离
- 曼哈顿距离
- 闵可夫斯基距离

课程入门阶段最常见的是 **欧氏距离**。

在二维平面中，两个点：

```text
A = (x1, y1)
B = (x2, y2)
```

它们的欧氏距离是：

```text
d(A, B) = sqrt((x1 - x2)^2 + (y1 - y2)^2)
```

这其实就是初中、高中几何里两点距离公式。

### 5. n维空间中的欧氏距离

机器学习里的样本通常不止两个特征。

如果一个样本有 n 个特征，就可以把它看成 n 维空间中的一个点，或者更准确地说，看成一个 n 维向量：

```text
x = [x1, x2, x3, ..., xn]
```

另一个样本记为：

```text
y = [y1, y2, y3, ..., yn]
```

那么它们之间的欧氏距离就是：

```text
d(x, y) = sqrt((x1 - y1)^2 + (x2 - y2)^2 + ... + (xn - yn)^2)
```

用求和符号写就是：

```text
d(x, y) = sqrt(Σ(xi - yi)^2), i = 1, 2, ..., n
```

### 6. 线性代数视角：欧氏距离就是向量长度

你提到“欧氏距离在 n 维空间的处理有一点线性代数的影子”，这个感觉是对的。

从线性代数角度看，一个样本就是一个向量。两个样本之间的差：

```text
x - y = [x1 - y1, x2 - y2, ..., xn - yn]
```

仍然是一个向量。

欧氏距离本质上就是这个“差向量”的长度，也叫 **二范数**：

```text
d(x, y) = ||x - y||2
```

如果用矩阵乘法的形式表示：

```text
||x - y||2 = sqrt((x - y)^T (x - y))
```

这里的 `(x - y)^T (x - y)` 可以理解为差向量和它自己的点积，也就是把每个维度上的差分别平方再相加。

所以 KNN 里的“距离”，其实就是在特征空间里比较两个样本向量之间的远近。

这也是为什么线性代数会在机器学习里频繁出现：机器学习经常把数据组织成向量和矩阵，然后用向量距离、点积、矩阵运算来表达样本之间的关系。

### 7. 一个重要提醒：特征量纲会影响距离

因为 KNN 依赖距离，所以不同特征的数值范围会明显影响结果。

例如：

```text
身高：1.60 ~ 1.90
收入：3000 ~ 30000
```

如果直接计算欧氏距离，收入这个特征的数值变化远大于身高，就可能在距离计算中占据主导地位。

因此使用 KNN 前，通常需要做特征预处理，例如：

- 标准化
- 归一化

这样不同特征才更容易在距离计算中公平地参与比较。

## 第二节 思路分析

KNN 主要可以解决两类问题：

- 分类问题
- 回归问题

这两类问题的前半部分思路基本一样，区别主要出现在最后一步：分类问题用投票，回归问题用均值。

### 1. 分类问题和回归问题

分类问题预测的是离散类别。

例如：

```text
电影类型：爱情片 / 动作片
鸢尾花类别：setosa / versicolor / virginica
是否患病：是 / 否
```

这些结果不是连续数值，而是一个个类别。对于这种问题，KNN 会让最近的 K 个邻居进行多数表决，哪个类别出现次数最多，就把它作为预测结果。

回归问题预测的是连续数值。

例如：

```text
房屋价格：85.6 万
气温：26.5 摄氏度
销售额：12000 元
```

这些结果是可以计算平均值的数值。对于这种问题，KNN 会取最近 K 个邻居的目标值，然后计算平均值，把这个平均值作为预测结果。

所以你说“只有数值问题才可以回归”这个理解基本是对的，更准确地说是：**回归问题的预测目标通常必须是连续数值型数据**。

### 2. KNN处理问题的共同步骤

无论是分类问题还是回归问题，KNN 的前几步都是一样的：

1. 准备已知样本数据。
2. 给定一个未知样本。
3. 计算未知样本和每一个已知样本之间的欧氏距离。
4. 按照欧氏距离从小到大进行升序排列。
5. 取出距离最近的 K 个样本。

可以简单表示为：

```text
未知样本 -> 计算距离 -> 距离升序排列 -> 取最近 K 个样本
```

完成这几步之后，再根据任务类型选择不同的预测方式。

### 3. 分类问题：多数表决

分类问题中，最近的 K 个样本都有各自的类别标签。

KNN 会统计这 K 个样本中每个类别出现的次数，然后选择出现次数最多的类别作为未知样本的预测值。

例如，当 `K = 5` 时，最近的 5 个邻居类别为：

```text
爱情片、爱情片、动作片、爱情片、动作片
```

统计结果是：

```text
爱情片：3 次
动作片：2 次
```

所以未知样本会被预测为：

```text
爱情片
```

这种方式就是多数表决。

### 4. 回归问题：计算均值

回归问题中，最近的 K 个样本对应的是数值型目标。

KNN 会把这 K 个样本的目标值取出来，然后计算它们的平均值，作为未知样本的预测值。

例如，当 `K = 3` 时，最近的 3 个邻居房价为：

```text
80 万、90 万、100 万
```

那么预测房价就是：

```text
(80 + 90 + 100) / 3 = 90 万
```

所以未知样本的预测结果是：

```text
90 万
```

### 5. 分类与回归的核心区别

KNN 的分类和回归在“找邻居”这一步是一样的，真正的区别在于如何根据邻居得到最终答案。

```text
分类问题：最近 K 个样本 -> 统计类别 -> 多数表决 -> 得到类别预测值
回归问题：最近 K 个样本 -> 统计数值 -> 计算均值 -> 得到连续数值预测值
```

因此可以记成：

```text
离散目标用投票，连续目标用均值。
```
