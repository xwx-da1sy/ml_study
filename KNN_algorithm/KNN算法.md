## 第一节 KNN算法简介

KNN，全称是 `K-Nearest Neighbors`，中文通常叫 **K 近邻算法**。

它是一种比较直观的机器学习算法：当我们要判断一个新样本属于哪一类时，不是先训练出一条复杂的公式，而是去训练集中找和它“最像”的若干个样本，然后根据这些邻居的结果来做判断。

一句话理解：

```text
看离我最近的 K 个样本大多数属于哪一类，我就把当前样本判断为哪一类。
```

例如判断一部电影是爱情片还是动作片，可以用“打斗镜头次数”和“接吻镜头次数”作为特征。现在来了一部新电影，我们就计算它和已有电影之间的距离，找出距离最近的 K 部电影。如果这 K 部电影里爱情片更多，那么新电影就更可能被判断为爱情片。

### 1. KNN的基本思想

KNN 的核心思想可以拆成三步：

1. 准备已经有标签的数据集。
2. 计算新样本和训练集中每个样本之间的距离。
3. 找出距离最近的 K 个样本，并根据它们的标签进行投票。

如果是分类问题，通常看 K 个邻居中哪个类别最多。

如果是回归问题，也可以取 K 个邻居目标值的平均值，得到一个连续数值预测结果。

### 2. K值的含义

这里的 `K` 表示参与判断的最近邻样本数量。

- `K = 1`：只看离新样本最近的 1 个样本，结果容易受噪声影响。
- `K` 较大：判断会更稳定，但也可能把距离较远、关系没那么强的样本算进来。

所以 KNN 中的 `K` 不是模型自动学出来的参数，而是我们需要提前设置的超参数。实际使用时，通常会通过测试集或交叉验证来选择一个表现较好的 K 值。K值是一个“超参数”。

### 3. K的选择与模型复杂度

K 的大小会直接影响 KNN 模型的复杂度。

这里说的“模型复杂度”，不是指 KNN 有很多参数或者公式很复杂。KNN 本身几乎没有训练过程，它的复杂度主要体现在：模型对训练数据局部细节的敏感程度。

当 `K` 过小时，例如 `K = 1`，模型只参考最近的一个样本。这样做会非常依赖训练集中的局部点。

如果训练集中刚好有噪声点、异常点，或者某个样本的位置比较特殊，模型就可能被这个单独的样本影响，做出不稳定的判断。

这时模型会把训练数据中的细节甚至噪声都记下来，决策边界往往会变得非常曲折。它在训练集上可能表现很好，但遇到新的测试样本时泛化能力差，这就是 **过拟合**。

可以理解为：

```text
K 过小 -> 太相信局部邻居 -> 对噪声敏感 -> 模型复杂 -> 容易过拟合
```

当 `K` 过大时，模型会参考大量邻居，甚至把离当前样本较远、关系不大的样本也算进来。

这样模型的判断会变得过于平滑，局部差异被忽略。它不太容易被个别噪声影响，但也可能学不到数据中真实存在的细节规律。

这时模型在训练集和测试集上都可能表现一般，属于 **欠拟合**。

可以理解为：

```text
K 过大 -> 参考范围太广 -> 忽略局部规律 -> 模型简单 -> 容易欠拟合
```

所以选择 K 值时，本质上是在做一个平衡：

- K 小：模型更灵活，方差较大，容易过拟合。
- K 大：模型更平滑，偏差较大，容易欠拟合。

实际使用时，通常不会只凭感觉选 K，而是尝试多个 K 值，然后通过验证集、测试集或交叉验证来比较模型效果。

### 4. 距离度量

KNN 之所以能判断“谁和谁更像”，关键就在于距离计算。

常见距离有：

- 欧氏距离
- 曼哈顿距离
- 闵可夫斯基距离

课程入门阶段最常见的是 **欧氏距离**。

在二维平面中，两个点：

```text
A = (x1, y1)
B = (x2, y2)
```

它们的欧氏距离是：

```text
d(A, B) = sqrt((x1 - x2)^2 + (y1 - y2)^2)
```

这其实就是初中、高中几何里两点距离公式。

### 5. n维空间中的欧氏距离

机器学习里的样本通常不止两个特征。

如果一个样本有 n 个特征，就可以把它看成 n 维空间中的一个点，或者更准确地说，看成一个 n 维向量：

```text
x = [x1, x2, x3, ..., xn]
```

另一个样本记为：

```text
y = [y1, y2, y3, ..., yn]
```

那么它们之间的欧氏距离就是：

```text
d(x, y) = sqrt((x1 - y1)^2 + (x2 - y2)^2 + ... + (xn - yn)^2)
```

用求和符号写就是：

```text
d(x, y) = sqrt(Σ(xi - yi)^2), i = 1, 2, ..., n
```

### 6. 线性代数视角：欧氏距离就是向量长度

你提到“欧氏距离在 n 维空间的处理有一点线性代数的影子”，这个感觉是对的。

从线性代数角度看，一个样本就是一个向量。两个样本之间的差：

```text
x - y = [x1 - y1, x2 - y2, ..., xn - yn]
```

仍然是一个向量。

欧氏距离本质上就是这个“差向量”的长度，也叫 **二范数**：

```text
d(x, y) = ||x - y||2
```

如果用矩阵乘法的形式表示：

```text
||x - y||2 = sqrt((x - y)^T (x - y))
```

这里的 `(x - y)^T (x - y)` 可以理解为差向量和它自己的点积，也就是把每个维度上的差分别平方再相加。

所以 KNN 里的“距离”，其实就是在特征空间里比较两个样本向量之间的远近。

这也是为什么线性代数会在机器学习里频繁出现：机器学习经常把数据组织成向量和矩阵，然后用向量距离、点积、矩阵运算来表达样本之间的关系。

### 7. 一个重要提醒：特征量纲会影响距离

因为 KNN 依赖距离，所以不同特征的数值范围会明显影响结果。

例如：

```text
身高：1.60 ~ 1.90
收入：3000 ~ 30000
```

如果直接计算欧氏距离，收入这个特征的数值变化远大于身高，就可能在距离计算中占据主导地位。

因此使用 KNN 前，通常需要做特征预处理，例如：

- 归一化：`(当前值 - 最小值) / (最大值 - 最小值)`
- 标准化：`(当前值 - 均值) / 标准差`

这样不同特征才更容易在距离计算中公平地参与比较。

## 第二节 思路分析

KNN 主要可以解决两类问题：

- 分类问题
- 回归问题

这两类问题的前半部分思路基本一样，区别主要出现在最后一步：分类问题用投票，回归问题用均值。

### 1. 分类问题和回归问题

分类问题预测的是离散类别。

例如：

```text
电影类型：爱情片 / 动作片
鸢尾花类别：setosa / versicolor / virginica
是否患病：是 / 否
```

这些结果不是连续数值，而是一个个类别。对于这种问题，KNN 会让最近的 K 个邻居进行多数表决，哪个类别出现次数最多，就把它作为预测结果。

回归问题预测的是连续数值。

例如：

```text
房屋价格：85.6 万
气温：26.5 摄氏度
销售额：12000 元
```

这些结果是可以计算平均值的数值。对于这种问题，KNN 会取最近 K 个邻居的目标值，然后计算平均值，把这个平均值作为预测结果。

所以你说“只有数值问题才可以回归”这个理解基本是对的，更准确地说是：**回归问题的预测目标通常必须是连续数值型数据**。

### 2. KNN处理问题的共同步骤

无论是分类问题还是回归问题，KNN 的前几步都是一样的：

1. 准备已知样本数据。
2. 给定一个未知样本。
3. 计算未知样本和每一个已知样本之间的欧氏距离。
4. 按照欧氏距离从小到大进行升序排列。
5. 取出距离最近的 K 个样本。

可以简单表示为：

```text
未知样本 -> 计算距离 -> 距离升序排列 -> 取最近 K 个样本
```

完成这几步之后，再根据任务类型选择不同的预测方式。

### 3. 分类问题：多数表决

分类问题中，最近的 K 个样本都有各自的类别标签。

KNN 会统计这 K 个样本中每个类别出现的次数，然后选择出现次数最多的类别作为未知样本的预测值。

例如，当 `K = 5` 时，最近的 5 个邻居类别为：

```text
爱情片、爱情片、动作片、爱情片、动作片
```

统计结果是：

```text
爱情片：3 次
动作片：2 次
```

所以未知样本会被预测为：

```text
爱情片
```

这种方式就是多数表决。

### 4. 回归问题：计算均值

回归问题中，最近的 K 个样本对应的是数值型目标。

KNN 会把这 K 个样本的目标值取出来，然后计算它们的平均值，作为未知样本的预测值。

例如，当 `K = 3` 时，最近的 3 个邻居房价为：

```text
80 万、90 万、100 万
```

那么预测房价就是：

```text
(80 + 90 + 100) / 3 = 90 万
```

所以未知样本的预测结果是：

```text
90 万
```

### 5. 分类与回归的核心区别

KNN 的分类和回归在“找邻居”这一步是一样的，真正的区别在于如何根据邻居得到最终答案。

```text
分类问题：最近 K 个样本 -> 统计类别 -> 多数表决 -> 得到类别预测值
回归问题：最近 K 个样本 -> 统计数值 -> 计算均值 -> 得到连续数值预测值
```

因此可以记成：

```text
离散目标用投票，连续目标用均值。
```

## 第三节 KNN解决分类问题代码实现.

### 1. 准备数据集

下面这段代码是在准备一个最简单的 KNN 分类数据集：

```python
x_train = [[0], [1], [2], [3]]
y_train = [0, 0, 1, 1]
x_test = [[5]]
```

其中：

- `x_train` 表示训练集中的特征数据。
- `y_train` 表示每个训练样本对应的分类标签。
- `x_test` 表示一个新的未知样本，用来让模型预测它属于哪一类。

对应关系可以写成：

```text
输入 x    标签 y
0         0
1         0
2         1
3         1
```

也就是说，训练数据告诉模型：

```text
x 是 0 或 1 时，类别是 0
x 是 2 或 3 时，类别是 1
```

所以当测试数据是：

```python
x_test = [[5]]
```

模型会根据训练集中已有样本的距离关系进行判断。因为 `5` 离 `2` 和 `3` 更近，按照这个简单规律，它更可能被预测为类别 `1`。

### 2. 为什么 x_train 要写成二维结构

这里容易误解的一点是：

```python
x_train = [[0], [1], [2], [3]]
```

它不是表示“有两个特征”，而是表示数据结构是二维的。

机器学习库通常要求输入特征 `X` 的格式是：

```text
[样本数, 特征数]
```

当前这个数据集的含义是：

```text
4 个样本，每个样本 1 个特征
```

所以它的形状是：

```text
4 x 1
```

如果写成表格，就是：

```text
样本编号    特征1
1           0
2           1
3           2
4           3
```

每个内部列表表示一个样本的特征向量：

```text
[0] 表示第 1 个样本，只有 1 个特征，特征值是 0
[1] 表示第 2 个样本，只有 1 个特征，特征值是 1
[2] 表示第 3 个样本，只有 1 个特征，特征值是 2
[3] 表示第 4 个样本，只有 1 个特征，特征值是 3
```

如果直接写成：

```python
x_train = [0, 1, 2, 3]
```

它就是一维结构，很多机器学习库无法明确判断这是“4 个样本，每个样本 1 个特征”，还是“1 个样本，4 个特征”，因此通常会要求写成二维形式。

### 3. 写成矩阵是什么样子

`x_train = [[0], [1], [2], [3]]` 写成矩阵是一个 4 行 1 列的矩阵：

```text
X_train =
| 0 |
| 1 |
| 2 |
| 3 |
```

也可以写成：

```text
X_train =
[
  [0],
  [1],
  [2],
  [3]
]
```

重点是：

```text
行表示样本
列表示特征
```

所以这个矩阵不是一行四列：

```text
[0 1 2 3]
```

而是四行一列：

```text
[0
 1
 2
 3]
```

### 4. 如果用 Java 表示

如果用 Java 写同样的数据集，可以写成二维数组：

```java
double[][] xTrain = {
    {0},
    {1},
    {2},
    {3}
};

int[] yTrain = {0, 0, 1, 1};

double[][] xTest = {
    {5}
};
```

对应关系是：

```text
xTrain[0] = {0}  -> yTrain[0] = 0
xTrain[1] = {1}  -> yTrain[1] = 0
xTrain[2] = {2}  -> yTrain[2] = 1
xTrain[3] = {3}  -> yTrain[3] = 1
```

如果每个样本有两个特征，才会写成：

```java
double[][] xTrain = {
    {0, 10},
    {1, 20},
    {2, 30},
    {3, 40}
};
```

此时才表示：

```text
4 个样本，每个样本 2 个特征
```

### 5. 归一化问题的模型实现

KNN 是根据距离判断“谁和谁更像”的算法，所以特征的数值范围会明显影响结果。

例如有两个特征：

```text
特征1：运动距离，范围大约是 1 ~ 5
特征2：消费金额，范围大约是 5000 ~ 12000
```

如果直接计算距离，`消费金额` 的数值范围远大于 `运动距离`，模型会更偏向消费金额这个特征，导致距离计算不公平。

这时通常要先做特征缩放，常见方式有：

- 归一化：把数据缩放到固定范围，例如 `0 ~ 1`。
- 标准化：把数据转换成均值为 `0`、标准差为 `1` 的分布。

在 KNN 中，实际更常用的是标准化。示例代码如下：

```python
from sklearn.neighbors import KNeighborsClassifier
from sklearn.preprocessing import StandardScaler

# 训练集特征
x_train = [
    [1.0, 5000],
    [2.0, 8000],
    [3.0, 10000],
    [4.0, 12000]
]

# 训练集标签
y_train = [0, 0, 1, 1]

# 待预测样本
x_test = [
    [3.5, 11000]
]

# 1. 创建标准化转换器
transfer = StandardScaler()

# 2. 对训练集进行标准化
x_train = transfer.fit_transform(x_train)

# 3. 对测试集进行同样的标准化
x_test = transfer.transform(x_test)

# 4. 创建 KNN 分类器
estimator = KNeighborsClassifier(n_neighbors=3)

# 5. 训练模型
estimator.fit(x_train, y_train)

# 6. 预测结果
y_predict = estimator.predict(x_test)

print(y_predict)
```

这里要注意：

- `fit_transform(x_train)`：先从训练集中学习标准化规则，再转换训练集。
- `transform(x_test)`：测试集只能使用训练集学到的标准化规则，不能重新 `fit`。
- 如果对测试集也使用 `fit_transform`，就相当于让测试集参与了规则学习，会造成数据泄漏。

如果使用归一化，也可以写成：

```python
from sklearn.preprocessing import MinMaxScaler

transfer = MinMaxScaler()
x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)
```

但是对 KNN 来说，入门阶段优先记住：

```text
KNN 依赖距离，所以通常需要先做特征缩放。
```

### 6. 归一化和标准化的算法公式

KNN 中常说的“归一化”，很多时候泛指“特征缩放”。严格一点说，常见做法可以分成下面几类。

#### 1. Min-Max 归一化

Min-Max 归一化会把数据缩放到固定范围，最常见的是 `[0, 1]`。

公式是：

```text
x_scaled = (x - x_min) / (x_max - x_min)
```

如果希望缩放到 `[a, b]`，公式是：

```text
x_scaled = a + (x - x_min) * (b - a) / (x_max - x_min)
```

对应 `scikit-learn`：

```python
from sklearn.preprocessing import MinMaxScaler

transfer = MinMaxScaler(feature_range=(0, 1))
x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)
```

它的特点是结果范围固定，容易理解；缺点是容易受极端值影响。比如收入特征里突然出现一个特别大的值，最大值会被拉高，其他样本会被压得很小。

#### 2. Z-score 标准化

标准化会把数据转换成均值为 `0`、标准差为 `1` 的形式。

公式是：

```text
x_scaled = (x - mean) / std
```

其中：

```text
mean = 这一列特征的平均值
std  = 这一列特征的标准差
```

对应 `scikit-learn`：

```python
from sklearn.preprocessing import StandardScaler

transfer = StandardScaler()
x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)
```

在 KNN 里，标准化通常比 Min-Max 归一化更常用。因为 KNN 关心距离，标准化可以让不同特征围绕自己的平均水平来比较，不会让数值范围大的特征天然占优势。

#### 3. MaxAbs 缩放

MaxAbs 缩放会用每一列特征的最大绝对值来缩放数据。

公式是：

```text
x_scaled = x / max(|x|)
```

对应 `scikit-learn`：

```python
from sklearn.preprocessing import MaxAbsScaler

transfer = MaxAbsScaler()
x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)
```

它会把数据缩放到 `[-1, 1]` 附近，并且不会破坏原来数据中的 `0`。如果数据本身比较稀疏，比如很多位置都是 `0`，这种方式比较合适。

#### 4. Robust 缩放

Robust 缩放使用中位数和四分位距，目的是减少异常值的影响。

公式可以理解为：

```text
x_scaled = (x - median) / IQR
```

其中：

```text
median = 中位数
IQR    = 第三四分位数 - 第一四分位数
```

对应 `scikit-learn`：

```python
from sklearn.preprocessing import RobustScaler

transfer = RobustScaler()
x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)
```

如果某一列特征里有明显异常值，例如收入里混入了极少数特别大的值，`RobustScaler` 往往比 `MinMaxScaler` 更稳。

#### 5. 向量归一化

前面几种方法通常是“按列处理”，也就是每一列特征各自缩放。向量归一化则通常是“按行处理”，它会把每个样本向量缩放成长度为 `1` 的向量。

公式是：

```text
x_scaled = x / ||x||
```

如果使用 L2 范数，就是：

```text
x_scaled = x / sqrt(x1^2 + x2^2 + ... + xn^2)
```

对应 `scikit-learn`：

```python
from sklearn.preprocessing import Normalizer

transfer = Normalizer(norm="l2")
x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)
```

这种方式更强调样本方向，而不是样本大小。它常见于文本向量、词频向量等场景。普通表格数据做 KNN 时，一般优先考虑 `StandardScaler` 或 `MinMaxScaler`。

#### 6. 训练集和测试集的正确处理方式

无论使用哪一种缩放算法，都要记住同一个原则：

```text
训练集：fit_transform
测试集：transform
```

也就是：

```python
transfer = StandardScaler()

x_train = transfer.fit_transform(x_train)
x_test = transfer.transform(x_test)
```

不能对测试集重新 `fit_transform`，因为测试集是用来模拟未知数据的。如果测试集也参与计算最小值、最大值、均值或标准差，就相当于提前看到了考试题的信息，会造成数据泄漏。

对于 KNN，可以先这样记：

```text
一般表格数据：优先 StandardScaler
数值范围明确且异常值少：可以 MinMaxScaler
异常值明显：可以 RobustScaler
稀疏数据：可以 MaxAbsScaler
文本向量或方向更重要的向量：可以 Normalizer
```

### 7. 归一化示例中的训练集矩阵

上面代码里的训练集是：

```python
x_train = [
    [1.0, 5000],
    [2.0, 8000],
    [3.0, 10000],
    [4.0, 12000]
]
```

它写成矩阵是：

```text
X_train =

          特征1     特征2
样本1      1.0      5000
样本2      2.0      8000
样本3      3.0      10000
样本4      4.0      12000
```

也就是：

```text
X_train =
[
  [1.0,  5000],
  [2.0,  8000],
  [3.0, 10000],
  [4.0, 12000]
]
```

重点规则是：

```text
行表示样本
列表示特征
```

所以：

- 横着看一行，是一个完整样本。
- 竖着看一列，是同一个特征在不同样本上的取值。

因此，“横着是样本”这个理解是对的。

“竖着是特征”要更准确地说成：

```text
每一列是一个特征。
每一列从上到下，是不同样本在这个特征上的取值。
```

例如：

```text
特征1这一列：1.0, 2.0, 3.0, 4.0
特征2这一列：5000, 8000, 10000, 12000
```

不能理解成“竖着的一整行是一个样本”。在 `scikit-learn` 中，标准格式始终是：

```text
X.shape = [样本数, 特征数]
```

当前例子中：

```text
X_train.shape = [4, 2]
```

含义是：

```text
4 个样本，每个样本 2 个特征
```

这段代码的核心作用是：

```text

再把 x_test 输入模型，
让模型根据距离最近的训练样本来判断 x_test 属于哪一类。
```

需要记住：

- `x_train` 是特征矩阵。
- `y_train` 是标签向量。
- `x_test` 是待预测样本。
- `[[0], [1], [2], [3]]` 是二维结构，形状是 `4 x 1`。
- 这里的二维不是指“两个特征”，而是指“样本数 x 特征数”的矩阵格式。
- KNN 依赖距离，多个特征的量纲差异较大时，通常要先做归一化或标准化。
- 在特征矩阵中，行表示样本，列表示特征。

## 第四节 其他距离计算方式

KNN 的核心是“根据距离找最近的 K 个邻居”。前面主要使用的是欧氏距离，但在机器学习中，距离不止一种。

常见的距离计算方式包括：

- 欧氏距离
- 曼哈顿距离
- 切比雪夫距离
- 闵可夫斯基距离

不同距离的本质区别是：如何把各个维度上的差异合并成一个最终距离值。

### 1. 曼哈顿距离

曼哈顿距离也叫 **城市街区距离**。

它的直观含义是：如果不能走斜线，只能沿着横向和纵向道路走，那么从一个点到另一个点需要走多远。

假设两个样本是：

```text
x = [x1, x2, x3, ..., xn]
y = [y1, y2, y3, ..., yn]
```

曼哈顿距离就是对应维度差值的绝对值求和：

```text
d(x, y) = |x1 - y1| + |x2 - y2| + ... + |xn - yn|
```

用求和符号写就是：

```text
d(x, y) = Σ |xi - yi|, i = 1, 2, ..., n
```

从线性代数角度看，曼哈顿距离就是差向量的 **一范数**：

```text
d(x, y) = ||x - y||1
```

### 2. 切比雪夫距离

切比雪夫距离关注的是所有维度差异中最大的那个。

假设两个样本是：

```text
x = [x1, x2, x3, ..., xn]
y = [y1, y2, y3, ..., yn]
```

切比雪夫距离就是对应维度差值的最大绝对值：

```text
d(x, y) = max(|x1 - y1|, |x2 - y2|, ..., |xn - yn|)
```

从线性代数角度看，切比雪夫距离就是差向量的 **无穷范数**：

```text
d(x, y) = ||x - y||∞
```

它表示两个样本在“差异最大的那个特征维度”上相差多少。

### 3. 闵可夫斯基距离

闵可夫斯基距离可以看成一类距离公式的总形式。

公式是：

```text
d(x, y) = (Σ |xi - yi|^p)^(1/p)
```

其中 `p` 是一个参数。

当 `p = 1` 时：

```text
d(x, y) = Σ |xi - yi|
```

这就是曼哈顿距离。

当 `p = 2` 时：

```text
d(x, y) = sqrt(Σ |xi - yi|^2)
```

这就是欧氏距离。

当 `p -> ∞` 时，它会逐渐接近只看最大维度差异的形式，也就是切比雪夫距离。

所以可以这样记：

```text
闵可夫斯基距离是一个统一公式。
p = 1 -> 曼哈顿距离
p = 2 -> 欧氏距离
p -> ∞ -> 切比雪夫距离
```

### 4. 在矩阵中的体现

在机器学习代码中，训练集通常不是一个单独向量，而是一个特征矩阵。

假设训练集矩阵是：

```text
X =
[
  [1, 1],
  [2, 3],
  [4, 5]
]
```

其中：

```text
行表示样本
列表示特征
```

待预测样本是：

```text
x_test = [3, 2]
```

计算距离时，本质上是让 `x_test` 分别和 `X` 中每一个训练样本做差。

差值矩阵可以写成：

```text
D = X - x_test

D =
[
  [1 - 3, 1 - 2],
  [2 - 3, 3 - 2],
  [4 - 3, 5 - 2]
]

D =
[
  [-2, -1],
  [-1,  1],
  [ 1,  3]
]
```

取绝对值后：

```text
|D| =
[
  [2, 1],
  [1, 1],
  [1, 3]
]
```

这个矩阵中：

- 每一行表示一个训练样本和 `x_test` 的差异。
- 每一列表示某一个特征维度上的差异。
- 距离计算就是把每一行的多个维度差异合并成一个数。

### 5. 曼哈顿距离的矩阵计算

曼哈顿距离是每一行绝对值求和：

```text
|D| =
[
  [2, 1],
  [1, 1],
  [1, 3]
]
```

所以：

```text
d1 =
[
  2 + 1,
  1 + 1,
  1 + 3
]

d1 = [3, 2, 4]
```

如果写成矩阵形式，可以理解为：

```text
d1 = |D| * 1
```

这里的 `1` 表示全 1 列向量：

```text
1 =
[
  1,
  1
]
```

也就是把每一行的各个特征差异加起来。

在 NumPy 中可以写成：

```python
distances = np.sum(np.abs(X - x_test), axis=1)
```

### 6. 切比雪夫距离的矩阵计算

切比雪夫距离是每一行取最大绝对值：

```text
|D| =
[
  [2, 1],
  [1, 1],
  [1, 3]
]
```

所以：

```text
d∞ =
[
  max(2, 1),
  max(1, 1),
  max(1, 3)
]

d∞ = [2, 1, 3]
```

在 NumPy 中可以写成：

```python
distances = np.max(np.abs(X - x_test), axis=1)
```

### 7. 闵可夫斯基距离的矩阵计算

闵可夫斯基距离是先对每个维度差异取绝对值，再做 `p` 次方，按行求和，最后开 `p` 次方根。

矩阵形式可以理解为：

```text
dp = (sum(|D|^p, axis=1))^(1/p)
```

如果 `p = 3`，那么：

```text
|D| =
[
  [2, 1],
  [1, 1],
  [1, 3]
]
```

先做 3 次方：

```text
|D|^3 =
[
  [8,  1],
  [1,  1],
  [1, 27]
]
```

再按行求和：

```text
sum(|D|^3, axis=1) = [9, 2, 28]
```

最后开 3 次方根：

```text
d3 = [9^(1/3), 2^(1/3), 28^(1/3)]
```

在 NumPy 中可以写成：

```python
p = 3
distances = np.sum(np.abs(X - x_test) ** p, axis=1) ** (1 / p)
```

### 8. 小结

对于两个样本向量 `x` 和 `y`：

```text
曼哈顿距离：d(x, y) = ||x - y||1
欧氏距离：  d(x, y) = ||x - y||2
切比雪夫距离：d(x, y) = ||x - y||∞
闵可夫斯基距离：d(x, y) = ||x - y||p
```

对于训练集矩阵 `X` 和待预测样本 `x_test`：

```text
先计算差值矩阵：D = X - x_test
再对 D 的每一行计算距离
最后得到一个距离向量
```

距离向量中的每一个值，对应 `x_test` 到一个训练样本的距离。KNN 会根据这个距离向量进行升序排序，然后取距离最近的 K 个样本。
