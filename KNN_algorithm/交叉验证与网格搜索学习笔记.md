## 2. 为什么需要交叉验证

只做一次训练集和测试集划分时，模型分数可能受到随机划分影响。

例如：

```python
train_test_split(X, y, test_size=0.2, random_state=22)
```

这只得到一次训练集和测试集。如果换一个 `random_state`，测试样本不同，准确率也可能变化。

交叉验证的思想是：

```text
把训练集分成 K 份
每次用其中 1 份做验证集，其余 K-1 份做训练集
重复 K 次
最后计算平均分
```

这样可以更稳定地估计模型在未知数据上的表现。

## 3. 五折交叉验证

如果 `cv=5`，表示五折交叉验证：

```text
第 1 次：第 1 份验证，其余训练
第 2 次：第 2 份验证，其余训练
第 3 次：第 3 份验证，其余训练
第 4 次：第 4 份验证，其余训练
第 5 次：第 5 份验证，其余训练
```

得到 5 个验证分数后取平均值：

```text
平均验证分数 = (score1 + score2 + score3 + score4 + score5) / 5
```

## 4. 网格搜索是什么

KNN 中的 `n_neighbors` 是超参数，也就是需要我们提前指定的参数。

之前写法：

```python
KNeighborsClassifier(n_neighbors=5)
```

这相当于手动指定 `K=5`。

网格搜索会让模型尝试多个候选值：

```python
param_grid = {
    "knn__n_neighbors": range(1, 21)
}
```

然后对每一个 K 值都做交叉验证，最后选择平均验证分数最高的 K。

## 5. 为什么要用 Pipeline

KNN 通常需要标准化，但交叉验证时不能先对整个训练集一次性标准化再划分折。

正确逻辑是：

```text
每一折内部：
当前折训练部分 -> fit 标准化器
当前折验证部分 -> transform
```

如果验证部分提前参与了均值和标准差的计算，就会造成数据泄露。

`Pipeline` 可以自动保证这个流程：

```python
pipeline = Pipeline([
    ("scaler", StandardScaler()),
    ("knn", KNeighborsClassifier()),
])
```

可以理解成：

```text
原始特征 X
-> 标准化
-> KNN 模型
-> 预测结果
```

## 7. 训练集、验证集、测试集的职责

```text
训练集：让模型学习
验证集：选择超参数
测试集：最终考试
```

注意：

```text
不能反复根据测试集结果调整 K。
```

如果这样做，测试集就不再是完全陌生的数据，最终评估会偏乐观。

## 8. 线性代数视角

鸢尾花特征矩阵是：

```text
X.shape = (150, 4)
```

意思是：

```text
150 行 -> 150 个样本
4 列   -> 4 个特征
```

交叉验证主要操作的是矩阵的行：

```text
每一折选择不同的样本行作为训练部分和验证部分
```

标准化主要操作的是矩阵的列：

```text
每一列特征减去均值，再除以标准差
```

KNN 预测主要使用样本行向量之间的距离：

```text
||x_test - x_i||_2
```

所以本节可以记成：

```text
交叉验证：分行
标准化：调列
KNN：算行向量之间的距离
```

这会自然衔接到线性回归：

```text
线性回归：用 X 的列做线性组合，得到预测向量 y_hat
```

也就是：

```text
y_hat = Xw
```
