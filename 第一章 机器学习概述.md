## 1. 基本概念

### 特征

特征是模型用来做判断或预测的输入信息。

例如预测房价时，面积、楼层、房龄、卧室数量都可以作为特征。

从线性代数角度看，一个样本的多个特征可以组成一个向量：

```text
x = [面积, 楼层, 房龄, 卧室数量]
```

### 标签

标签是我们希望模型学习并预测的目标结果。

例如预测房价时，房价就是标签。

训练数据可以理解为：

```text
特征 x -> 标签 y
```

### 样本、特征和标签的命名

- `feature`：特征，对应输入数据 `x` 或 `X`。注意不是 `future`。
- `target` / `label`：标签，对应目标值 `y`。
- `sample`：样本，一条完整数据。比如一套房子的面积、楼层、房龄和房价就是一个样本。

通常用大写 `X` 表示所有样本的特征矩阵，用小写 `y` 表示所有样本的标签向量：

```text
X = 所有样本的特征
y = 所有样本的标签
```

### 训练集和测试集

数据通常会拆成训练集和测试集：

- `train_set`：训练集，用来让模型学习规律。
- `test_set`：测试集，用来检查模型在没见过的数据上表现如何。

常见划分比例是 `8:2` 或 `7:3`，也就是 80%/70% 用来训练，20%/30% 用来测试。`8:3` 一般不作为常见比例，可能是把 `8:2` 或 `7:3` 记混了。

拆分后常见变量名：

```text
X_train = 训练集的特征
X_test  = 测试集的特征
y_train = 训练集的标签
y_test  = 测试集的标签
```

训练时模型使用 `X_train` 和 `y_train`；测试时把 `X_test` 输入模型得到预测结果，再和 `y_test` 对比。

### 算法学习的两种方式

- 规则：硬编码地告诉计算机如何从特征得到标签，常见形式是 `if-else`。
- 数据：通过大量特征和标签，让计算机自己学习特征与标签之间的关系。线性回归就是基于数据学习模型的例子。

### 线性回归

线性回归是一种用于预测连续数值的监督学习算法。

单个特征时：

```text
y = wx + b
```

多个特征时：

```text
y = w1x1 + w2x2 + ... + wnxn + b
```

线性回归的目标是从数据中学习合适的权重 `w` 和偏置 `b`，让预测值尽量接近真实标签。

## 2. 机器学习算法分类

机器学习算法常按“训练数据有没有标签、学习方式是什么”来分类。

### 有监督学习

有监督学习使用带标签的数据训练模型，也就是每个样本都有 `X` 和对应答案 `y`。

- 分类问题：预测离散类别。例如根据邮件内容判断“垃圾邮件 / 正常邮件”。
- 回归问题：预测连续数值。例如根据面积、房龄、楼层预测房价。

### 无监督学习

无监督学习使用没有标签的数据，模型需要自己发现数据中的结构。

- 聚类问题：把相似样本自动分组。例如根据消费习惯把用户分成几类人群。

### 半监督学习

半监督学习同时使用少量有标签数据和大量无标签数据。它适合“数据很多，但人工打标签很贵”的场景。

例如有 10 万张商品图片，全部人工标注成本很高；如果只标注其中 5000 张，再让模型利用剩下未标注图片中的相似性和分布规律辅助学习，就能减少人工标记量，所以可以大幅降低标记成本。

它的核心直觉是：相似的样本通常有相似的标签。无标签数据虽然没有答案 `y`，但仍然有特征 `X`，可以帮助模型理解数据分布。常见做法是“伪标签”：先用少量有标签数据训练模型，再让模型给无标签数据预测标签，把高置信度结果当作临时标签继续训练。风险是如果伪标签错了，错误可能被继续放大。

### 强化学习

强化学习不是直接给标准答案，而是让智能体在环境中不断尝试，根据奖励学习更好的行动策略。

- `agent`：智能体，做决策的人或程序。
- `action`：动作，智能体可以做的选择。
- `environment`：环境，智能体行动所在的世界。
- `reward`：奖励，环境对行动结果的反馈。

例如训练游戏 AI：`agent` 是游戏角色，`action` 是移动或攻击，`environment` 是游戏地图，`reward` 可以是得分增加、通关或失败惩罚。

这一节的核心不是马上记住所有算法，而是先建立判断入口：看到问题时，先问“有没有标签？要预测类别、数值，还是发现数据分组？是否需要通过奖励来学习行动？”

## 3. 模型训练流程

参照黑马程序员课程的讲法，机器学习项目通常可以按下面这条主线推进：

```text
获取数据 -> 数据处理 -> 特征工程 -> 机器学习 -> 模型评估 -> 模型预测
```

- 获取数据：拿到原始数据，例如表格、图片、文本、日志等。
- 数据处理：清洗数据，处理缺失值、异常值、重复值，并划分训练集和测试集。
- 特征工程：把原始数据整理成模型更容易学习的特征 `X`，这是机器学习里很关键的一步。
- 机器学习：选择合适的算法，用 `X_train` 和 `y_train` 训练模型。
- 模型评估：用 `X_test` 得到预测结果，并和 `y_test` 对比，判断模型效果。
- 模型预测：当模型效果可接受后，用它对新的未知数据进行预测。

简单理解：前面几步是在准备高质量数据，中间一步是在训练模型，后面几步是在检查模型并真正使用模型。

## 4. 特征工程入门

### 什么是特征

特征是描述样本的信息，是模型的输入 `X`。例如预测房价时，面积、楼层、房龄就是特征；预测邮件是否垃圾邮件时，邮件中的词语、发送时间、发件人信息也可以是特征。

### 什么是特征工程

特征工程是把原始数据转换成更适合模型学习的特征的过程。机器学习不只是选算法，很多时候模型效果好不好，和特征处理得好不好关系很大。

特征工程常见内容包括：特征提取、特征预处理、特征降维、特征选择、特征组合等。

### 特征提取

特征提取是从原始数据中提取可用于建模的信息。比如把文本转换成词频向量，把图片转换成像素或图像特征，把日期拆成年份、月份、星期几。

### 特征预处理

特征预处理是对已有特征做格式或数值上的整理，让模型更容易训练。

- 归一化：把数据缩放到固定范围，常见是 `[0, 1]`。适合把不同量纲的数据放到相近尺度。
- 标准化：把数据转换成均值为 0、标准差为 1 的分布形式。常用于线性模型、距离类模型等。

简单区分：归一化看最大值和最小值，标准化看均值和标准差。

### 特征降维

特征降维是把高维特征压缩成低维特征，减少冗余信息和计算量。例如原来有 100 个特征，降到 20 个更重要的综合特征。常见方法有 PCA。

### 特征选择

特征选择是从已有特征中挑出更有用的特征，去掉无关或影响不大的特征。例如预测房价时，面积可能很重要，房屋编号可能没什么意义。

### 特征组合

特征组合是把多个特征组合成新的特征，让模型更容易发现关系。例如用“总价 / 面积”得到单价，用“房间数 / 面积”描述空间密度。

### 是否影响原始数据

概念上，特征工程通常不应该直接破坏原始数据，而是基于原始数据生成一份处理后的特征数据 `X`。但在代码里，如果直接对原表原地修改或覆盖保存，就会影响原始数据。

- 一般不改变原始数据：训练集/测试集划分、特征选择、特征降维、模型训练、模型评估、模型预测。这些步骤通常是生成新的数据子集、特征矩阵或模型结果。
- 可能改变原始数据：数据清洗、缺失值填充、异常值处理、归一化、标准化、特征组合。如果直接在原始表上修改，会改变原数据；如果先复制一份再处理，就不会影响原始数据。

实践中建议保留原始数据，只在副本或新的特征表上做处理，方便回溯和对比。

一句话理解：特征工程就是想办法把原始数据变成更好用的 `X`，让模型更容易从中学习规律。

## 5. 拟合、泛化与模型复杂度

### 什么是拟合

拟合可以理解为模型学习训练数据规律的过程。模型训练后，要同时看它在训练集和测试集上的表现。

```text
训练集表现 = 模型对学过的数据掌握得怎么样
测试集表现 = 模型对没见过的数据预测得怎么样
```

### 欠拟合

欠拟合是模型太简单，连训练集规律都没有学好。

```text
训练集表现不好，测试集表现也不好
```

例如用一条直线去拟合明显弯曲的数据，模型表达能力不够，训练和测试都会差。

### 过拟合

过拟合是模型把训练集学得太细，甚至把噪声也记住了，导致面对新数据时表现变差。

```text
训练集表现好，测试集表现不好
```

例如模型把每个训练样本都“背下来”，训练时很准，但遇到新样本就容易判断错。

### 恰好拟合

恰好拟合是模型既学到了主要规律，又没有过度记忆训练数据。

```text
训练集表现好，测试集表现也好
```

这是我们希望达到的状态。

### 泛化

泛化指模型对未知数据的适应能力。模型不是只要在训练集上高分就好，更重要的是在测试集、真实新数据上也能表现稳定。

简单说：泛化能力强，说明模型真的学到了规律；泛化能力差，可能只是记住了训练数据。

### 奥卡姆剃刀原则

奥卡姆剃刀原则可以理解为：在效果差不多的情况下，优先选择更简单的模型。

放到机器学习里，就是不要为了追求训练集分数而把模型做得过于复杂。模型太复杂更容易过拟合；合适的简单模型往往更容易泛化。
